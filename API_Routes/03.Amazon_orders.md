# 📦 Amazon E-commerce API Documentation

## 🛠 1. Order Management (CRUD) 

*Core operations for individual order processing.*

1. `GET /api/v1/orders` - Fetch all orders (paginated).
2. `GET /api/v1/orders/:orderId` - Get full details of a specific order.
3. `POST /api/v1/orders` - Place a new order.
4. `PUT /api/v1/orders/:orderId` - Update an entire order record.
5. `PATCH /api/v1/orders/:orderId/status` - Update only the status (e.g., Shipped to Delivered).
6. `DELETE /api/v1/orders/:orderId` - Remove an order record.
7. `GET /api/v1/orders/exists/:orderId` - Validate if an OrderID is valid.

---

## 🔍 2. Advanced Search & Customer Filtering

*Routes focused on finding specific transactions and customer history.*

8. `GET /api/v1/orders/search?q=CustomerName` - Search orders by customer name.
9. `GET /api/v1/orders/customer/:customerId` - Retrieve all orders placed by a specific customer.
10. `GET /api/v1/orders/product/:productId` - Get all transactions for a specific product.
11. `GET /api/v1/orders/filter/category?name=Electronics` - Filter by product category.
12. `GET /api/v1/orders/filter/brand?name=BrightLux` - Filter by specific brand.
13. `GET /api/v1/orders/filter/status?type=Cancelled` - List all orders with a specific status.
14. `GET /api/v1/orders/filter/payment?method=Debit Card` - Filter by payment type.
15. `GET /api/v1/orders/location/country/:country` - Filter orders by country.
16. `GET /api/v1/orders/location/state/:state` - Filter orders by state.
17. `GET /api/v1/orders/location/city/:city` - Filter orders by city.
18. `GET /api/v1/orders/date-range?start=2023-01-01&end=2023-03-31` - Filter by order date.
19. `GET /api/v1/orders/high-value?threshold=500` - List orders where `TotalAmount` exceeds a limit.

## 📈 3. Financial Analytics & Business Intelligence 

*Routes for calculating revenue, taxes, and shipping costs.*

20. `GET /api/v1/reports/total-revenue` - Sum of all `TotalAmount` values.
21. `GET /api/v1/reports/total-tax` - Sum of all tax collected.
22. `GET /api/v1/reports/shipping-costs` - Total expenditure on shipping.
23. `GET /api/v1/reports/average-order-value` - Calculate the mean price of orders.
24. `GET /api/v1/reports/discounts-given` - Total amount lost to discounts.
25. `GET /api/v1/reports/by-category` - Revenue breakdown per category (e.g., Books vs Electronics).
26. `GET /api/v1/reports/by-payment-method` - Count of orders per payment type.
27. `GET /api/v1/reports/monthly-sales?year=2023` - Sales performance month-by-month.
28. `GET /api/v1/reports/top-customers` - Top 10 customers by total spend.
29. `GET /api/v1/reports/top-sellers` - Rank sellers by total items sold.
30. `GET /api/v1/reports/refund-rate` - Percentage of orders with "Returned" or "Cancelled" status.

## ✨ Good to Have (Bonus Features)

* **Financial Precision:** Since you are dealing with prices, ensure your backend uses `Decimal` or `Cent` integers to avoid floating-point errors (e.g., `319.86` should be handled carefully).
* **Audit Logs:** Create a table/collection to log whenever an `OrderStatus` changes (who changed it and when).
* **PDF Invoicing:** Use a library like `pdfkit` to generate a real PDF invoice for route #32.
* **Authorization Levels:** Practice role-based access. Only "Admin" should access `/reports`, while "Customers" can only access their own orders.
* **Search Indexing:** If using a database, ensure fields like `CustomerName` and `OrderID` are indexed for fast searching.
